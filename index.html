Ai-je besoin de tests unitaires ?

Oui ! Même dans ton cas, même si ton code te semble simple, parce que :

  • La dernière fois que tu ne l’as pas fait, ton code a provoqué un bug en production ;
  • Un jour, tu regretteras de ne pas l’avoir fait ;
  • Si tu ne le fais pas, il est possible que tu sois maudit sur 50 générations par ceux qui reprendront ton code ;
  • Peut-être que l’écriture du test te permettra de trouver une meilleure solution au problème que tu cherches à résoudre ;
  • Lorsque tu écris un test unitaire, tu documentes ton code ;
  • Écrire un test, c’est faire du code SOLID ;
  • Ne pas écrire un test, c’est comme utiliser un préservatif troué pour faire l’amour avec quelqu’un que l’on vient juste de rencontrer ;
  • Ne pas le faire, c’est comme jouer à la roulette russe ;
  • Conduire dans l’obscurité par temps de brouillard sans feu allumé est une pure folie ;
  • Le diable se cache dans les détails ;
  • C’est à cause des développeurs qui n’écrivent pas de test que Donald Trump est actuellement le président des USA ;
  • Un chaton meurt dès qu’un développeur écrit du code sans écrire le test correspondant ;
  • C’est parce que le Monde n’avait pas de tests unitaires que les licornes ont disparu ;
  • Et cette liste n’est pas exhaustive, si tu souhaites l’enrichir, n’hésite pas !

D’après une idée d’Éric Daspet un jour de mai 2017.
